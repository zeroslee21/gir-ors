# GIR ORS 아키텍처 개요

## 시스템 아키텍처 요약

GIR ORS는 314개 한국 온실가스 감축 방법론을 탐색하는 정적 웹 애플리케이션입니다. 프로덕션 배포 시 서버 런타임 없이 사전 렌더링된 HTML 파일로만 동작하여 서버 비용을 제거합니다.

### 핵심 특성

**정적 사이트 생성 (SSG)**
- Next.js 15 with `output: "export"` 설정
- `next build` 명령 실행 시 모든 페이지를 사전 생성
- 배포 대상: 정적 HTML 파일만 필요

**데이터 독립성**
- 방법론 정보, 섹터, 통계는 JSON 파일로 관리
- 데이터 업데이트 시 빌드만 다시 실행하면 됨
- 데이터베이스 서버 불필요

**인터랙티브한 사용자 경험**
- React 클라이언트 컴포넌트로 검색, 필터링, 북마크 구현
- localStorage를 통한 영구 저장소
- 테마 변경 (라이트/다크 모드)

## 설계 원칙

### 1. 서버 비용 0
- 정적 파일만 배포
- AWS EC2 + Nginx로 간단한 정적 파일 서빙
- 자동 확장 불필요

### 2. 데이터 독립성
- 관계형 데이터베이스 불필요
- JSON 데이터와 소스 코드 함께 버전 관리
- 배포와 데이터 업데이트가 동시 진행

### 3. 높은 성능
- CDN 배포 최적화
- 코드 분할 (Code Splitting) 자동 수행
- 이미지 최적화 (Next.js Image Component)

### 4. 단순한 배포
- 정적 파일만 필요하므로 배포 간단
- ALB와 Nginx만으로 충분
- 무상태 애플리케이션

## 기술 스택

### 프론트엔드
- **Next.js 15**: React 기반 메타프레임워크
- **TypeScript**: 타입 안전성
- **Tailwind CSS**: 유틸리티 기반 스타일링
- **Recharts**: 차트 라이브러리 (방법론 연간 통계)
- **Fuse.js**: 클라이언트 사이드 검색 (314개 방법론)
- **next-themes**: 라이트/다크 모드 관리

### 빌드 및 배포
- **Node.js 20**: 런타임 환경
- **npm**: 패키지 관리자
- **AWS EC2 + ALB**: 호스팅
- **Nginx**: 정적 파일 서빙

### 개발 도구
- **ESLint**: 코드 품질 관리
- **Prettier**: 코드 포매팅
- **TypeScript**: 정적 타입 검사

## 배포 아키텍처

```
사용자 브라우저
    ↓
AWS ALB (Application Load Balancer)
    ↓
AWS EC2 인스턴스 (Nginx)
    ↓
정적 HTML/CSS/JS 파일
```

### 배포 흐름

1. **개발 단계**: 로컬에서 `npm run dev` 실행
2. **빌드 단계**: `npm run build` 실행하여 `.next/static` 디렉토리 생성
3. **배포 단계**: 빌드 결과를 EC2 서버에 업로드
4. **서빙 단계**: Nginx가 정적 파일을 클라이언트에게 직접 서빙

### Nginx 설정

```
- 포트 80에서 HTTP 요청 수신
- `/` 경로를 `.next/static` 디렉토리로 매핑
- 캐시 헤더 설정 (캐시 효율성 증대)
- gzip 압축 활성화
```

## 빌드 프로세스

### 빌드 단계

1. **정적 생성 (Static Generation)**
   - `getStaticParams()` 호출로 모든 방법론 페이지 생성
   - 각 섹터 페이지도 정적으로 생성

2. **번들링 (Bundling)**
   - 모든 JavaScript/CSS를 번들로 변환
   - 동적 import로 자동 코드 분할

3. **최적화 (Optimization)**
   - 이미지 최적화
   - 폰트 로딩 최적화
   - 번들 크기 최적화

4. **산출물 (Output)**
   - `.next/static` 디렉토리의 정적 파일들
   - HTML, CSS, JavaScript 모두 포함

## 성능 고려사항

### 빌드 시간
- 314개 페이지 생성: 약 30-60초
- 전체 빌드: 약 1-2분

### 번들 크기
- 메인 JavaScript: ~100KB (gzip)
- 각 페이지별 추가 청크: ~10-20KB
- 이미지 및 폰트: ~200KB

### 런타임 성능
- 페이지 로딩 시간: < 1초
- 검색 응답 시간: < 100ms
- 필터링 응답 시간: < 50ms

- **섹터 범위**: 다수
